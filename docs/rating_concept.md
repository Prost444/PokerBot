# Концепция рейтинговой системы покерного клуба ШАД

## 1. Диагностика проблемы

Когда игра идёт на фишки без последствий, у игроков нет стимула принимать взвешенные решения. Разорение за две раздачи не несёт никаких издержек — можно просто взять новые фишки или уйти. Это порождает каскад проблем: опытным игрокам скучно играть против хаотичных олл-инов, новички не учатся, а посещаемость падает.

Нужна система, которая создаёт **репутационные последствия** каждого решения за столом, при этом оставаясь дружественной, не превращая клуб в казино.

## 2. Обзор исследованных подходов

### 2.1. Очковые лиговые системы (League Points)

Используются в барных и клубных лигах (World Tavern Poker, Atlanta Poker Club, The Nuts Poker League). Суть: за каждый вечер начисляются очки в зависимости от занятого места. Типичная формула:

```
Points(k, N) = base + (N - k) * step + bonus_final_table
```

Плюсы: простота, понятность. Минусы: не учитывают силу соперников, не адаптируются к уровню игрока, слабо различают удачу и мастерство.

### 2.2. Global Poker Index (GPI)

Профессиональная система ранжирования 450 000+ турнирных игроков. Каждый результат оценивается как произведение трёх факторов:

```
Score = FPF × BIF × AF
```

Где FPF — процент обыгранных игроков, BIF — фактор бай-ина (логарифмический), AF — фактор старения результата. Итоговый рейтинг — сумма лучших 5 результатов за каждый полугодовой период.

Плюсы: учёт старения результатов, масштабируемость. Минусы: заточен под турниры с бай-ином, сложно адаптировать к бесплатному формату.

### 2.3. Адаптированная система Эло для мультиплеерного покера

Классическая система Эло (шахматы) обобщена для N игроков. Ключевая идея: каждая сессия — это набор попарных сравнений между всеми участниками.

```
S_i = (N - k_i) / (N - 1)       — фактический результат игрока i (место k_i из N)
E_i = avg(1 / (1 + 10^((R_j - R_i) / 400)))   — ожидаемый результат на основе рейтингов
R_new = R_old + K × (S_i - E_i)  — обновление рейтинга
```

Плюсы: самокорректирующаяся, учитывает силу поля, проверена десятилетиями. Минусы: чувствительна к выбору K-фактора, требует достаточного числа сессий для стабилизации.

### 2.4. Баунти-фишки

В турнирах с баунти часть призового фонда «привязана» к каждому игроку. Выбил соперника — забрал его баунти-фишку. Существуют варианты: фиксированные баунти, прогрессивные (часть баунти побеждённого добавляется к твоему), mystery bounty (случайные награды).

Плюсы: добавляют азарт и дополнительные цели. Минусы: могут провоцировать охоту за слабыми игроками. В бесплатном формате ценность баунти нужно привязывать к рейтинговым очкам.

## 3. Предлагаемая система: «ШАД Покер Рейтинг»

Гибридная система, сочетающая **адаптированный Эло** как ядро с **баунти-бонусами** и **сезонной структурой** для долгосрочной вовлечённости.

### 3.1. Структура вечера

Каждый четверг проводится одна или несколько турнирных сессий. В зависимости от количества игроков:

| Игроков | Столов | Формат |
|---------|--------|--------|
| 10 или менее | 1 | Один финальный стол |
| 11–20 | 2 | Два стола → финальный стол (топ-5 с каждого) |
| 21–30 | 3 | Три стола → финальный стол (топ-3–4 с каждого) |

Каждый игрок получает одинаковый стартовый стек (например, 5000 фишек). **Ребаев нет** — это ключевое правило, которое заставляет ценить каждую фишку.

### 3.2. Рейтинг Эло: ядро системы

**Начальный рейтинг:** 1200 для всех новых игроков.

**После каждой сессии:**

1. Все N участников ранжируются по итоговому количеству фишек (или порядку выбывания).
2. Фактический результат игрока i с позицией k:

```
S_i = (N - k) / (N - 1)
```

Примеры: 1-е из 10 → S = 1.0, 5-е из 10 → S = 0.556, 10-е из 10 → S = 0.0.

3. Ожидаемый результат — среднее попарных вероятностей:

```
E_i = (1 / (N - 1)) × Σ_{j≠i} [1 / (1 + 10^((R_j - R_i) / 400))]
```

4. Обновление рейтинга:

```
R_new = R_old + K × (N - 1) × (S_i - E_i)
```

Множитель (N-1) нормализует масштаб обновлений при разном числе участников.

**К-фактор (скорость изменения рейтинга):**

| Условие | K |
|---------|---|
| Первые 10 сессий игрока | 40 |
| Рейтинг < 1400 | 30 |
| Рейтинг ≥ 1400 | 20 |

Высокий K для новичков позволяет быстро определить их реальный уровень. Затем рейтинг стабилизируется.

### 3.3. Баунти-система: «Скальпы»

Это та часть, о которой ты слышал, и она отлично работает как дополнение к Эло. В нашем бесплатном формате баунти-фишки не имеют денежного эквивалента — вместо этого они конвертируются в **рейтинговые бонусы**.

**Механика:**

- Каждый игрок при рассадке получает одну именную баунти-фишку (отличающуюся цветом от игровых).
- Когда игрок выбивает другого (забирает все его фишки), он получает баунти-фишку выбитого.
- В конце вечера баунти конвертируются в бонус к Эло:

```
bounty_bonus = Σ (2 + (R_выбитого - R_охотника) / 200)
```

То есть выбить игрока сильнее себя — ценнее (до +4 к рейтингу за скальп), а выбить слабого — почти ничего не даёт (+1..+2). Это **защищает новичков** от целенаправленной охоты и поощряет отвагу против сильных.

Минимальный бонус за скальп — 1 очко (нельзя уйти в минус за баунти).

**Почему это нормализует поведение:** безрассудный олл-ин означает, что ты рискуешь отдать свою баунти-фишку, а с ней — рейтинговые очки другому игроку. Это создаёт «стоимость» проигрыша, которой не хватало в свободной игре.

### 3.4. Бонус за регулярность: «Верность столу»

Чтобы поощрять постоянное посещение:

```
attendance_multiplier = min(1.0 + 0.05 × streak, 1.25)
```

Где streak — количество подряд посещённых четвергов (макс. бонус ×1.25 за 5+ недель подряд). Множитель применяется к основному изменению Эло (но только к положительным изменениям). Пропуск обнуляет серию.

Это мягкий стимул: не наказывает за пропуск, но награждает за регулярность.

### 3.5. Рассадка по столам

При нескольких столах рассадка делается по принципу «змейки» (snake draft) по рейтингу:

1. Отсортировать игроков по Эло.
2. Распределить по столам: 1→A, 2→B, 3→C, 4→C, 5→B, 6→A, 7→A, 8→B, ...

Это обеспечивает **баланс столов** по силе: на каждом столе примерно равный средний рейтинг. Сильные и слабые игроки перемешаны.

Альтернативный вариант для особых вечеров: «дивизионная» рассадка (сильные с сильными), чтобы дать новичкам поиграть в комфортной среде. Но в штатном режиме змейка предпочтительнее — она учит новичков быстрее.

### 3.6. Сезонная структура

Каждый **сезон = 8 недель** (≈ 2 месяца). Это удобно ложится на учебные периоды.

**В течение сезона:**
- Рейтинг Эло кумулятивный, не сбрасывается.
- Ведётся таблица лидеров с текущим рейтингом.
- Для сезонного зачёта учитываются **лучшие 6 из 8 вечеров** (можно пропустить 2 без потерь — для учёбы/болезни).

**В конце сезона:**
- Топ-3 по Эло — «Чемпионы сезона» (символические награды: кубок, грамота, особое место за следующим финальным столом).
- Лучший прогресс (наибольший рост Эло за сезон) — «Прорыв сезона».
- Больше всего скальпов за сезон — «Охотник сезона».

**Между сезонами:** Эло НЕ обнуляется, но применяется мягкий регресс к среднему:

```
R_new_season = R_old × 0.8 + 1200 × 0.2
```

Это даёт отстающим надежду, а лидерам — вызов. Рейтинг 1500 станет 1440, рейтинг 1000 станет 1040.

## 4. Пример расчёта

Вечер: 10 игроков. Игрок Алексей (Эло 1350) занимает 2-е место и выбил 2 игроков (Эло 1180 и 1420).

**Основной Эло:**
- S = (10 - 2) / (10 - 1) = 0.889
- E = вычисляется попарно со всеми 9 соперниками (допустим, E = 0.62 на основе рейтингов)
- K = 20 (рейтинг < 1400, но > 10 сессий — нет, рейтинг < 1400 → K = 30)
- ΔR = 30 × 9 × (0.889 - 0.62) = 30 × 9 × 0.269 = **+72.6**

**Баунти-бонус:**
- Скальп 1 (Эло 1180): 2 + (1180 - 1350)/200 = 2 + (-0.85) = 1.15 → округлим до 1.2
- Скальп 2 (Эло 1420): 2 + (1420 - 1350)/200 = 2 + 0.35 = 2.35
- Итого баунти: **+3.55**

**Регулярность:** Алексей ходит 3-й четверг подряд → множитель = 1 + 0.05×3 = 1.15
- Применяется к основному ΔR: 72.6 × 1.15 = **83.5**

**Итого:** Алексей получает +83.5 + 3.55 ≈ **+87 Эло**. Новый рейтинг: 1437.

## 5. Что требует учёта в системе

| Элемент | Данные для ввода |
|---------|-----------------|
| Регистрация игрока | Имя / никнейм |
| Начало вечера | Дата, список участников |
| Рассадка | Автоматическая по рейтингу (змейка) |
| Ход турнира | Порядок выбывания, кто кого выбил |
| Конец вечера | Финальный ранжир (по фишкам или порядку выбывания) |
| Расчёт | Автоматический пересчёт Эло, баунти, множителей |

## 6. Почему именно эта комбинация

1. **Эло как ядро** — проверена 60+ лет в шахматах, Go, киберспорте. Самокорректируется: завышенный рейтинг быстро падает при плохих результатах. Для аудитории математиков и CS-специалистов — естественный и понятный выбор.

2. **Баунти как стимул** — создаёт ценность каждой раздачи. Безрассудный олл-ин = высокий риск потерять свой скальп, что ударит по рейтингу. При этом баунти привязаны к разнице рейтингов, что защищает новичков.

3. **Отсутствие ребаев** — одна жизнь за вечер. Это главный «нормализатор» поведения: каждая фишка стоит дорого, потому что новых не будет.

4. **Сезоны с мягким сбросом** — не дают рейтингу «окостеневать», поддерживают интригу, дают новичкам шанс. Формула лучших 6 из 8 снимает давление обязательного посещения.

5. **Змейка при рассадке** — балансирует столы, предотвращает ситуации «стол смерти» и «стол для слабых».
